# Coreth Atomik İşlem Formatı

Bu sayfanın, `Coreth`'deki atomik işlemleri nasıl serileştirdiğimize dair gerçeğin tek kaynağı olması amaçlanır. Bu doküman paketleme için [ilkel serileştirme](serialization-primitives.md) ve kriptografik kullanıcı kimliği için [secp256k1](cryptographic-primitives.md#cryptography-in-the-avalanche-virtual-machine) formatı kullanır.

## Codec ID'si

Bazı verilerin önüne verinin nasıl serileştireceğini belirten bir codec ID \(unt16\) eklenmiştir. Şimdilik tek geçerli codec ID 0'dır \(`0x00 0x00`\).

## Girdiler

Coreth Atomik İşlemlerinin girdileri ya bu zincirden gelen bir `EVMInput`ya da başka bir zincirden gelen \(bir  `SECP256K1TransferInput`eren\) bir bil`TransferableInput`ir. `EVMInput`, bu zincirden fonlar harcamak için `ExportTx`'de kullanılırken, `TransferableInput`başka bir zincirden atomik UTXO'ları içe aktarmak için kullanılacaktır.

### EVM Girdisi

Fonların düşüleceği bir EVM hesabını `ExportTx`'in bir parçası olarak belirleyen girdi tipidir.

#### EVM Girdisi Neleri İçerir

Bir EVM Girdisi bir `address`, `amount`, `assetID`ve `nonce` içerir.

* **`Address`**, fonlar hangi EVM adresinden transfer edilecekse o EVM'nin adresidir.
* **`Amount`**, transfer edilecek varlığın tutarıdır \(AVAX için nAVAX cinsinden ve diğer tüm varlıklar için en küçük denomination \(değer birimi\) olarak belirtilir\).
* **`AssetID`**, transfer edilecek varlığın kimliğidir.
* **`Nonce`**, fonların dışa aktarıldığı EVM hesabının nonce'udur.

#### Gantt EVM Girdisi Spesifikasyonu

```text
+----------+----------+-------------------------+
| address  : [20]byte |                20 bytes |
+----------+----------+-------------------------+
| amount   : uint64   |                08 bytes |
+----------+----------+-------------------------+
| asset_id : [32]byte |                32 bytes |
+----------+----------+-------------------------+
| nonce    : uint64   |                08 bytes |
+----------+----------+-------------------------+
                      |                68 bytes |
                      +-------------------------+
```

#### Proto EVM Girdisi Spesifikasyonu

```text
message  {
    bytes address = 1; // 20 bytes
    uint64 amount = 2; // 08 bytes
    bytes assetID = 3; // 32 bytes
    uint64 nonce = 4;  // 08 bytes
}
```

### Transfer Edilebilir Girdi

Transfer Edilebilir Girdi bir `SECP256K1TransferInput`'yi wrap eder. Transfer edilebilir girdiler, belli bir transfer girdisine sahip özel UTXO'ları tanımlar.

#### Transfer Edilebilir Girdi Neleri İçerir

Transfer edilebilir bir girdi bir `TxID`, `UTXOIndex`, `AssetID` ve `Input`içerir.

* **`TxID`**, bu girdinin hangi işlemden bir çıktıyı tükettiğini tanımlayan 32 baytlık bir dizilimdir.
* **`UTXOIndex`**, bu girdinin belirtilen işlemde hangi utxo'yu tükettiğini tanımlayan bir tamsayıdır.
* **`AssetID`**, bu girdinin referans verdiği varlığı tanımlayan 32 baytlık bir dizilimdir.
* **`Input`**, aşağıda tanımlanan bir `SECP256K1TransferInput` ögesidir.

#### Gantt Transfer Edilebilir Girdi Özellikleri

```text
+------------+----------+------------------------+
| tx_id      : [32]byte |               32 bytes |
+------------+----------+------------------------+
| utxo_index : int      |               04 bytes |
+------------+----------+------------------------+
| asset_id   : [32]byte |               32 bytes |
+------------+----------+------------------------+
| input      : Input    |      size(input) bytes |
+------------+----------+------------------------+
                        | 68 + size(input) bytes |
                        +------------------------+
```

#### Proto Transfer Edilebilir Girdi Özellikleri

```text
message TransferableInput {
    bytes tx_id = 1;       // 32 bytes
    uint32 utxo_index = 2; // 04 bytes
    bytes asset_id = 3;    // 32 bytes
    Input input = 4;       // size(input)
}
```

### SECP256K1 Transfer Girdisi

Secp265k1 transfer girdisi, harcanmamış bir [secp256k1](./cryptographic-primitives#cryptography-in-the-avalanche-virtual-machine) transfer çıktısını harcamaya olanak tanır.

#### SECP256K1 Transfer Çıktısı Neleri İçerir

Bir secp256k1 transfer girdisi, bir `Amount`ve `AddressIndices` içerir.

* **`TypeID`**, bu girdi tipinin kimliğidir.  `0x00000005`değerindedir.
* **`Amount`**, bu girdinin UTXO'dan harcıyor olması gereken miktarı belirten bir long'tur. Pozitif bir değer olmalıdır. UTXO'da belirlenen miktara eşit olmalıdır.
* **`AddressIndices`**, UTXO'yu harcamak için kullanılan özel anahtarları tanımlayan benzersiz bir tamsayılar \(int\) listesidir. Her UTXO'da UTXO'yu harcayabilecek adresler dizisi bulunur. Her tamsayı, işlemi imzalayacak bu adres dizisindeki endeksi temsil eder. Dizi en düşükten en yükseğe doğru sıralanmalıdır.

#### Gantt SECP256K1 Transfer Girdisi Özellikleri

```text
+-------------------------+-------------------------------------+
| type_id         : int   |                             4 bytes |
+-----------------+-------+-------------------------------------+
| amount          : long  |                             8 bytes |
+-----------------+-------+-------------------------------------+
| address_indices : []int |  4 + 4 * len(address_indices) bytes |
+-----------------+-------+-------------------------------------+
                          | 16 + 4 * len(address_indices) bytes |
                          +-------------------------------------+
```

#### Proto SECP256K1 Transfer Girdisi Özellikleri

```text
message SECP256K1TransferInput {
    uint32 typeID = 1;                   // 04 bytes
    uint64 amount = 2;                   // 08 bytes
    repeated uint32 address_indices = 3; // 04 bytes + 04 bytes * len(address_indices)
}
```

#### SECP256K1 Transfer Girdisi Örneği

Şununla bir ödeme girdisi yapalım:

* **`TypeId`**: 5
* **`Amount`**: 500000000000
* **`AddressIndices`**: [0]

```text
[
    TypeID         <- 0x00000005
    Amount         <- 500000000000 = 0x000000746a528800,
    AddressIndices <- [0x00000000]
]
=
[
    // type id:
    0x00, 0x00, 0x00, 0x05,
    // amount:
    0x00, 0x00, 0x00, 0x74, 0x6a, 0x52, 0x88, 0x00,
    // length:
    0x00, 0x00, 0x00, 0x01,
    // sig[0]
    0x00, 0x00, 0x00, 0x00,
]
```

## Çıktılar

Coreth Atomik İşlemlerine giden çıktılar ya bu zincirdeki bir adresin bakiyesine eklenecek bir `EVMOutput` ögesidir ya da başka bir zincire taşınacak \(bir `SECP256K1TransferOutput` içeren\) bir '`TransferableOutput` ögesidir.

EVM Çıktısı, bu zincire fonlar eklemek için `ImportTx`'de kullanılırken, `TransferableOutput`başka bir zincire atomik UTXO'lar aktarmak için kullanılacaktır.

### EVM Çıktısı

Bir EVM hesabına uygulanacak bir durum değişikliğini `ImportTx`'nin bir parçası olarak belirleyen bir çıktı tipidir.

#### EVM Çıktısı Neleri İçerir

Bir EVM Çıktısı bir `address`, `amount` ve `assetID` içerir.

* **`Address`**, fonları alacak olan EVM adresidir.
* **`Amount`**, transfer edilecek varlığın tutarıdır \(AVAX için nAVAX cinsinden ve diğer tüm varlıklar için en küçük denomination \(değer birimi\) olarak belirtilir\).
* **`AssetID`**, transfer edilecek varlığın kimliğidir.

#### Gantt EVM Çıktısı Spesifikasyonu

```text
+----------+----------+-------------------------+
| address  : [20]byte |                20 bytes |
+----------+----------+-------------------------+
| amount   : uin64    |                08 bytes |
+----------+----------+-------------------------+
| asset_id : [32]byte |                32 bytes |
+----------+----------+-------------------------+
                      |                60 bytes |
                      +-------------------------+
```

#### Proto EVM Çıktısı Spesifikasyonu

```text
message  {
    bytes address = 1; // 20 bytes
    uint64 amount = 2; // 08 bytes
    bytes assetID = 3; // 32 bytes
}
```

### Transfer Edilebilir Çıktı

Transfer edilebilir çıktılar bir `SECP256K1TransferOutput`'yi bir varlık kimliği ile wrap eder.

#### Transfer Edilebilir Çıktı İçeriği

Transfer edilebilir bir çıktı, bir `AssetID` ve bir `SECP256K1TransferOutput` olan bir `Output` içerir.

* **`AssetID`**, bu çıktının hangi varlığa referans verdiğini belirten 32 baytlık bir dizilimdir.
* **`Output`**, aşağıda tanımlanan bir `SECP256K1TransferOutput` ögesidir.

#### Gantt Transfer Edilebilir Çıktı Özellikleri

```text
+----------+----------+-------------------------+
| asset_id : [32]byte |                32 bytes |
+----------+----------+-------------------------+
| output   : Output   |      size(output) bytes |
+----------+----------+-------------------------+
                      | 32 + size(output) bytes |
                      +-------------------------+
```

#### Proto Transfer Edilebilir Çıktı Özellikleri

```text
message TransferableOutput {
    bytes asset_id = 1; // 32 bytes
    Output output = 2;  // size(output)
}
```

### SECP256K1 Transfer Çıktısı

Bir [secp256k1](cryptographic-primitives.md#cryptography-in-the-avalanche-virtual-machine) transfer çıktısı, belirlenen bir unix zamanından sonra bir varlığın belli bir miktarını, bir dizi adrese göndermeye olanak tanır.

#### SECP256K1 Transfer Çıktısı Neleri İçerir

Bir secp256k1 transfer çıktısı bir `TypeID`, `Amount`, `Locktime`, `Threshold` ve `Addresses` içerir.

* **`TypeID`**, bu çıktı tipinin kimliğidir.  `0x00000007`değerindedir.
* **`Amount`**, bu çıktının sahip olduğu varlığın miktarını belirten bir long'tur \(uzun veri tipi\). Pozitif bir değer olmalıdır.
* **`Locktime`**, bu çıktının sonrasında harcanabileceği unix zaman damgasını içeren bir long'tur. Unix zaman damgası saniyeye kadar kesin bir birimdir.
* **`Threshold`**, çıktıyı harcamak için gereken benzersiz imzaların sayısını belirten bir tamsayıdır. **`Addresses`** uzunluğundan küçük veya ona eşit olmalıdır. **`Addresses`** ögesi boş ise, 0 olmalıdır.
* **`Addresses`**, bu çıktıyı harcamak için kullanılabilecek özel anahtarlara karşılık gelen benzersiz adreslerin bir listesidir. Adresler sözlük sıralaması ile sıralanmalıdır.

#### Gantt SECP256K1 Transfer Çıktısı Özellikleri

```text
+-----------+------------+--------------------------------+
| type_id   : int        |                        4 bytes |
+-----------+------------+--------------------------------+
| amount    : long       |                        8 bytes |
+-----------+------------+--------------------------------+
| locktime  : long       |                        8 bytes |
+-----------+------------+--------------------------------+
| threshold : int        |                        4 bytes |
+-----------+------------+--------------------------------+
| addresses : [][20]byte |  4 + 20 * len(addresses) bytes |
+-----------+------------+--------------------------------+
                         | 28 + 20 * len(addresses) bytes |
                         +--------------------------------+
```

#### Proto SECP256K1 Transfer Çıktısı Özellikleri

```text
message SECP256K1TransferOutput {
    uint32 typeID = 1;            // 04 bytes
    uint64 amount = 2;            // 08 bytes
    uint64 locktime = 3;          // 08 bytes
    uint32 threshold = 4;         // 04 bytes
    repeated bytes addresses = 5; // 04 bytes + 20 bytes * len(addresses)
}
```

## Atomik İşlemler

Atomik İşlemler zincirler arasında fonlar taşımak için kullanılır. İki tipi vardır: `ImportTx` ve `ExportTx`.

### ExportTx

ExportTx, Coreth'ten farklı bir zincire fonlar aktarma işlemidir.

#### ExportTx Neleri İçerir

Bir ExportTx \(Dışa Aktarma İşlemi\) bir `typeID`, `networkID`, `blockchainID`, `destinationChain`, `inputs` ve `exportedOutputs` içerir.

* **`typeID`**, bir ExportTx \(Dışa Aktarma İşlemi\) tipi olan bir tamsayıdır. Bir exportTx için tip kimliği 1'dir.
* **`networkID`**, bu işlemin gönderilmesinin amaçlandığı Avalanche ağını tanımlayan bir tamsayıdır. Bu kimlik mainnet'i, fuji'yi, vb. ifade ediyor olabilir ve EVM'nin ağ kimliğinden farklıdır.
* **`blockchainID`**, bu işlemin gönderildiği blok zinciri tanımlayan 32 baytlık bir dizilimidir.
* **`destinationChain`**, bu işlemle fonların dışa aktarıldığı blok zinciri tanımlayan 32 baytlık bir dizilimdir.
* **`inputs`**, Dışa Aktarma İşleminin fonlanmasına yönelik bir EVM Girdileri dizilimidir.
* **`exportedOutputs`**, `destinationChain` ögesine aktarılacak bir Transfer Edilebilir Çıktılar dizilimidir.

#### Gantt ExportTx Spesifikasyonu

```text
+---------------------+----------------------+-------------------------------------------------+
| typeID              : int                  |                                        04 bytes |
+---------------------+----------------------+-------------------------------------------------+
| networkID           : int                  |                                        04 bytes |
+---------------------+----------------------+-------------------------------------------------+
| blockchainID        : [32]byte             |                                        32 bytes |
+---------------------+----------------------+-------------------------------------------------+
| destinationChain    : [32]byte             |                                        32 bytes |
+---------------------+----------------------+-------------------------------------------------+
| inputs              : []EvmInput           |                          4 + size(inputs) bytes |
+---------------------+----------------------+-------------------------------------------------+
| exportedOutputs     : []TransferableOutput |                 4 + size(exportedOutputs) bytes |
+----------+----------+----------------------+-------------------------------------------------+
                                             | 80 + size(inputs) + size(exportedOutputs) bytes |
                                             +-------------------------------------------------+
```

Şimdi bir EVM Çıktısı oluşturalım:

* **`TypeID`**: `1`
* **`NetworkID`**: `12345`
* **`BlockchainID`**: `0x91060eabfb5a571720109b5896e5ff00010a1cfe6b103d585e6ebf27b97a1735`
* **`DestinationChain`**: `0xd891ad56056d9c01f18f43f58b5c784ad07a4a49cf3d1f11623804b5cba2c6bf`
* **`Inputs`**:
   * `"Example EVMInput as defined above"`
* **`Exportedoutputs`**:
   * `"Example TransferableOutput as defined above"`

```text

    NetworkID        <- 0x00003039
    BlockchainID     <- 0x91060eabfb5a571720109b5896e5ff00010a1cfe6b103d585e6ebf27b97a1735
```

### ImportTx

ImportTx, başka bir zincirden Coreth'e fonlar aktarılması işlemdir.

#### ImportTx Neleri İçerir

Bir ImportTx \(İçe Aktarma İşlemi\) bir `typeID`, `networkID`, `blockchainID`, `destinationChain`, `importedInputs` ve `Outs` içerir.

* **`typeID`**, bir ImportTx \(İçe Aktarma İşlemi\) tipini belirten bir tamsayıdır. Bir `ImportTx`'in tip kimliği 0'dır.
* **`networkID`**, bu işlemin gönderilmesinin amaçlandığı Avalanche ağını tanımlayan bir tamsayıdır. Bu kimlik mainnet'i, fuji'yi, vb. ifade ediyor olabilir ve EVM'nin ağ kimliğinden farklıdır.
* **`blockchainID`**, bu işlemin gönderildiği blok zinciri tanımlayan 32 baytlık bir dizilimidir.
* **`sourceChain`**, fonların hangi blok zincirden aktarılacağını tanımlayan 32 baytlık bir dizilimdir.
* **`importedInputs`**, İçe Aktarma İşlemini fonlamaya yönelik Transfer Edilebilir Girdilerin bir dizilimidir.
* **`Outs`**, bu zincire aktarılacak bir EVM Çıktıları dizilimidir.

#### Gantt ImportTx Spesifikasyonu

```text
+---------------------+----------------------+-------------------------------------------------+
| typeID              : int                  |                                        04 bytes |
+---------------------+----------------------+-------------------------------------------------+
| networkID           : int                  |                                        04 bytes |
+---------------------+----------------------+-------------------------------------------------+
| blockchainID        : [32]byte             |                                        32 bytes |
+---------------------+----------------------+-------------------------------------------------+
| sourceChain         : [32]byte             |                                        32 bytes |
+---------------------+----------------------+-------------------------------------------------+
| importedInputs      : []TransferableInput  |                  4 + size(importedInputs) bytes |
+---------------------+----------------------+-------------------------------------------------+
| outs                : []EVMOutput          |                            4 + size(outs) bytes |
+----------+----------+----------------------+-------------------------------------------------+
                                             |    80 + size(importedInputs) + size(outs) bytes |
                                             +-------------------------------------------------+
```

## Kimlik Bilgileri

Kimlik bilgilerinin bir olası tipi vardır: `SECP256K1Credential`. Her bir kimlik bilgisi bir Girdi ile eşleşir. Kimlik bilgilerinin sırası, girdilerin sırasıyla uyumludur.

### SECP256K1 Kimlik Bilgisi

[Secp256k1](./cryptographic-primitives#cryptography-in-the-avalanche-virtual-machine) kimlik bilgisi 65 baytlık kurtarılabilir imzaların bir listesini içerir.

#### SECP256K1 Kimlik Bilgisi İçeriği

* **`TypeID`**, bu tipin kimliğidir.  `0x00000009`değerindedir.
* **`Signatures`**, 65 baytlık bir geri getirilebilir imzalar dizilimidir. İmzaların sırası, girdinin imza indeksleriyle eşleşmelidir.

#### Gantt SECP256K1 Kimlik Bilgisi Özellikleri

```text
+------------------------------+---------------------------------+
| type_id         : int        |                         4 bytes |
+-----------------+------------+---------------------------------+
| signatures      : [][65]byte |  4 + 65 * len(signatures) bytes |
+-----------------+------------+---------------------------------+
                               |  8 + 65 * len(signatures) bytes |
                               +---------------------------------+
```

#### Proto SECP256K1 Kimlik Bilgisi Özellikleri

```text
message SECP256K1Credential {
    uint32 typeID = 1;             // 4 bytes
    repeated bytes signatures = 2; // 4 bytes + 65 bytes * len(signatures)
}
```

## İmzalanmış İşlem

İmzalanmış bir işlem, imzalanmamış bir `AtomicTx` ve kimlik bilgileri içerir.

### İmzalanmış İşlem İçeriği

Bir İmzalanmış işlem bir `CodecID`, `AtomicTx`ve `Credentials` içerir.

* **`CodecID`** Tek güncel geçerli codec kimliği `00 00`'dır.
* **`AtomicTx`**, yukarıda tanımlanan bir atomik işlemdir.
* **`Credentials`**, bir kimlik bilgileri dizilimidir. Her bir kimlik bilgisi AtomicTx'te aynı indeksteki girdiye karşılık gelir.

### Gantt İmzalanmış İşlem Özellikleri

```text
+---------------------+--------------+------------------------------------------------+
| codec_id            : uint16       |                                        2 bytes |
+---------------------+--------------+------------------------------------------------+
| atomic_tx           : AtomicTx     |                          size(atomic_tx) bytes |
+---------------------+--------------+------------------------------------------------+
| credentials         : []Credential |                    4 + size(credentials) bytes |
+---------------------+--------------+------------------------------------------------+
                                     |   6 + size(atomic_tx) + len(credentials) bytes |
                                     +------------------------------------------------+
```

### Proto İmzalanmış İşlem Özellikleri

```text
message Tx {
    uint16 codec_id = 1;                 // 2 bytes
    AtomicTx atomic_tx = 2;              // size(atomic_tx)
    repeated Credential credentials = 3; // 4 bytes + size(credentials)
}
```

## UTXO

UTXO, bir işlem çıktısının bağımsız bir temsilidir.

### UTXO İçeriği

Bir UTXO, bir `CodecID`, `TxID`, `UTXOIndex`, `AssetID` ve `Output` içerir.

* **`CodecID`** Tek geçerli `CodecID` şudur: `00 00`
* **`TxID`**, 32 baytlık bir işlem kimliğidir. İşlem ID'leri imzalı işlemin sha256 kadar baytını alarak hesaplanır.
* **`UTXOIndex`**, , bu utxo'nun yaratıldığı **`TxID`** vasıtasıyla belirlenen işlem çıktısını belirten bir tamsayıdır.
* **`AssetID`** bu utxo'nun hangi varlığa referans yaptığını belirten 32 baytlık bir dizilimdir.
* **`Output`**, bu utxo'yu yaratan çıktı nesnesidir. Çıktıların serileştirilmesi yukarıda tanımlanmıştı.

### Gantt UTXO Spesifikasyonu

```text
+--------------+----------+-------------------------+
| codec_id     : uint16   |                 2 bytes |
+--------------+----------+-------------------------+
| tx_id        : [32]byte |                32 bytes |
+--------------+----------+-------------------------+
| output_index : int      |                 4 bytes |
+--------------+----------+-------------------------+
| asset_id     : [32]byte |                32 bytes |
+--------------+----------+-------------------------+
| output       : Output   |      size(output) bytes |
+--------------+----------+-------------------------+
                          | 70 + size(output) bytes |
                          +-------------------------+
```

### Proto UTXO Spesifikasyonu

```text
message Utxo {
    uint16 codec_id = 1;     // 02 bytes
    bytes tx_id = 2;         // 32 bytes
    uint32 output_index = 3; // 04 bytes
    bytes asset_id = 4;      // 32 bytes
    Output output = 5;       // size(output)
}
```
