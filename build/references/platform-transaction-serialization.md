# プラットフォームトランザクションフォーマット

このファイルは、Avalancheのプラットフォームバーチャルマシンで取引をどのようにシリアライズするか、という真実の単一のソース`Platform Chain`であることを意味します。`P-Chain`このドキュメントは、パッキングに[プリミティブシリアライズ](serialization-primitives.md)形式を使用し、暗号ユーザ識別には[secp256k1](cryptographic-primitives.md#secp-256-k1-addresses)を使用します。

## コーデックID

一部のデータは、コーデックID（unt16）でプリペンドされています。つまり、データがどのようにデスリアライズされるかを示します。現在のところ、有効なコーデックIDは0 \(\)です`0x00 0x00`。

## トランスファブル出力

トランスファー可能な出力は、アセットIDで出力をラップします。

### トランスファブル出力が含まれているもの

トランスファブル出力には、an`AssetID`と。`Output`

* **`AssetID`**は、この出力参照のアセットを定義する32バイト配列です。有効な唯一の場合、AVAX`AssetID`です。`AssetID`
* **`Output`**は、以下に定義したように出力です。たとえば、SECP256K1トランスファ出力となる可能性があります。

### Ganttトランスファブル出力仕様

```text
+----------+----------+-------------------------+
| asset_id : [32]byte |                32 bytes |
+----------+----------+-------------------------+
| output   : Output   |      size(output) bytes |
+----------+----------+-------------------------+
                      | 32 + size(output) bytes |
                      +-------------------------+
```

### Proto Transferable Output Specification

```text
message TransferableOutput {
    bytes asset_id = 1; // 32 bytes
    Output output = 2;  // size(output)
}
```

## SECP256K1トランスファー出力

[secp256k1](cryptographic-primitives.md#secp-256-k1-addresses)転送出力により、指定されたunix時間後に数量のアセットをコレクションに送信することができます。有効なアセットのみがAVAXです。

### **SECP256K1トランスファー出力が含まれているもの**

`Threshold`secp256k1トランスファ出力には、a `TypeID`, `Amount`, `Locktime`,、とが含まれています`Addresses`。

* **`TypeID`**iss は、この出力タイプのIDです。それは`0x00000007`。
* **`Amount`**は、この出力が所有するアセットの数を指定する長い陽性でなければなりません。
* **`Locktime`**は、この出力が後に使用できるunixタイムスタンプが含まれている長いです。unixタイムスタンプは、第二に固有です。
* **`Threshold`**iss is is an implementation is an implementation is an implementation.その長さに相当するものでなければなりません**`Addresses`**。空**`Addresses`**の場合、0でなければなりません。
* **`Addresses`**isは、この出力を費やすために使用できる秘密鍵に相当するユニークアドレスのリストです。アドレスは、辞書でソートする必要があります。

### **Gantt SECP256K1トランスファー出力仕様**

```text
+-----------+------------+--------------------------------+
| type_id   : int        |                        4 bytes |
+-----------+------------+--------------------------------+
| amount    : long       |                        8 bytes |
+-----------+------------+--------------------------------+
| locktime  : long       |                        8 bytes |
+-----------+------------+--------------------------------+
| threshold : int        |                        4 bytes |
+-----------+------------+--------------------------------+
| addresses : [][20]byte |  4 + 20 * len(addresses) bytes |
+-----------+------------+--------------------------------+
                         | 28 + 20 * len(addresses) bytes |
                         +--------------------------------+
```

### **Proto SECP256K1トランスファー出力仕様**

```text

    uint32 type_id = 1;           // 04 bytes
    uint64 amount = 2;            // 08 bytes

```

## 未署名 BaseTx

### **ベースTxが含まれているもの**

`NetworkID``BlockchainID``Outputs``Inputs``Memo`ベースtxには`TypeID`、a , ,と.

* **`TypeID`**iss は、このタイプのIDです。それは`0x00000000`。

```text

```

## 認証情報

認証情報は、1つの可能なタイプがあります。 `SECP256K1Credential`.各認証情報は、入力あるいはオペレーションとペアリングされます。認証情報の順番は、入力あるいはオペレーションの順番に一致します。

## SECP256K1

[secp256k1](cryptographic-primitives.md#secp-256-k1-addresses)認証情報には、65バイトで回復可能な署名のリストが含まれます。

### **SECP256K1認証情報が含まれているもの**

* **`TypeID`**iss は、このタイプのIDです。それは`0x00000009`。
* **`Signatures`**is 65バイト回復可能な署名の配列です。署名の順番は、入力の署名指標と一致する必要があります。

### **Gantt SECP256K1**

```text
+------------------------------+---------------------------------+
| type_id         : int        |                         4 bytes |
+-----------------+------------+---------------------------------+
| signatures      : [][65]byte |  4 + 65 * len(signatures) bytes |
+-----------------+------------+---------------------------------+
                               |  8 + 65 * len(signatures) bytes |
                               +---------------------------------+
```

### **Proto SECP256K1**

```text
message SECP256K1Credential {
    uint32 TypeID = 1;             // 4 bytes
    repeated bytes signatures = 2; // 4 bytes + 65 bytes * len(signatures)
}
```

## 署名されたトランザクション

署名トランザクションは、一連の認証情報が追加される未署名トランザクションです。

### 署名されたトランザクションが含まれているもの

`UnsignedTx`署名されたトランザクションには、a `CodecID`,と`Credentials`。

* **`CodecID`**現在の有効なコーデックidは唯一です。`00 00`
* **`UnsignedTx`**は、上記に記したように、署名されていないトランザクションです。
* **`Credentials`**iss は、認証情報の配列です。各認証情報は、この認証情報で同じインデックス内の入力とペアリングされます。

### Gantt署名されたトランザクション仕様

```text
+---------------------+--------------+------------------------------------------------+
| codec_id            : uint16       |                                        2 bytes |
+---------------------+--------------+------------------------------------------------+
| unsigned_tx         : UnsignedTx   |                        size(unsigned_tx) bytes |
+---------------------+--------------+------------------------------------------------+
| credentials         : []Credential |                    4 + size(credentials) bytes |
+---------------------+--------------+------------------------------------------------+
                                     | 6 + size(unsigned_tx) + len(credentials) bytes |
                                     +------------------------------------------------+
```

### Proto署名されたトランザクション仕様

```text
message Tx {
    uint32 codec_id = 1;                 // 2 bytes
    UnsignedTx unsigned_tx = 2;          // size(unsigned_tx)
    repeated Credential credentials = 3; // 4 bytes + size(credentials)
}
```

## UTXO

UTXOは、トランザクション出力をスタンドアロンで表したものです。

### UTXOが含まれている

`UTXOIndex``Output`UTXOには、a `CodecID`, `TxID`,、と。

* **`CodecID`**現在の有効なコーデックidは唯一です。`00 00`
* **`TxID`**は32バイトのトランザクションIDトランザクションIDは、署名されたトランザクションのバイト数で、sha256をとって計算されます。
* **`UTXOIndex`**isinで、このutxoが指定したトランザクションでどの出力が指定され**`TxID`**たかを指定します。
* **`AssetID`**は、このutxo参照アセットを定義する32バイト配列です。
* **`Output`**は、このutxoを作成した出力オブジェクトアウトプットのシリアリゼーションは上記の定義となりました。

#### Gantt UTXO<a id="gantt-utxo-specification"></a>

```text
+--------------+----------+-------------------------+
| codec_id     : uint16   |                 2 bytes |
+--------------+----------+-------------------------+
| tx_id        : [32]byte |                32 bytes |
+--------------+----------+-------------------------+
| output_index : int      |                 4 bytes |
+--------------+----------+-------------------------+
| asset_id     : [32]byte |                32 bytes |
+--------------+----------+-------------------------+
| output       : Output   |      size(output) bytes |
+--------------+----------+-------------------------+
                          | 70 + size(output) bytes |
                          +-------------------------+
```

### Proto UTXO

```text
message Utxo {
    uint32 codec_id = 1;     // 02 bytes
    bytes tx_id = 2;         // 32 bytes
    uint32 output_index = 3; // 04 bytes
    bytes asset_id = 4;      // 32 bytes
    Output output = 5;       // size(output)
}
```
